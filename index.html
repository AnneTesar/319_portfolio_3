Scalable Vector Graphics (SVG)

Scalable Vector Graphics, or SVGs, are a great way to store images. They are smaller to store and won't lose resolution when scaled. There are many neat effects that can be applied to SVGs. They can also be animated in a variety of ways. SVGs are growing in popularity; you can expect to see them more widely used in the future. This will introduce you to some simple ways to create and animate SVGs.

The Basics


A shape can be drawn by specifying position and size. Color can be specified in a variety of ways, as can any style element.

<svg width="200" height="200" >
	<circle cx="50" cy="50" r="40" style="fill:rgb(150,0,255)" stroke="black" stroke-width="4" />
</svg >

We can create a variety of shapes, including custom polygons. For these, we specify a series of points to draw between.

<svg width="200" height="200" >
	<polygon points="15,15 30,80 80,150 150,45" style="fill:yellow;stroke:grey;stroke-width:2">
</svg >

We can also specify individual points for Paths. These use functions like Move To (M), Line To (L), Bezier Curve (q), and Close (Z) to draw shapes. These shapes can be filled in or not. Here we are also demonstrating that you can put multiple shapes into the same SVG tag. 

<svg width="200" height="200" >
	<path d="M 0 150 q 100 -200 200 0" stroke="blue" stroke-width="5" fill="none" />
	<path d="M 0 175 L 100 100 L 200 175 Z" stroke="blue" stroke-width="3" fill="rgb(100, 150, 200)" /> 
</svg >

The order that tags are in determines their order drawn on the page. The pink circle was there first, then the green circle was placed on top of it. The green circle has an opacity of 0.5 so you can still see the pink under it. 

<svg width="200" height="200" >
	<circle cx="50" cy="50" r="40" fill="pink" />
	<circle cx="75" cy="75" r="40" opacity="0.5" fill="green" /> 
</svg >

Basic Animation with SMIL


This animate tag inside the rectangle tag applies the animation to that rectangle. We can specify things like starting and ending positions, duration of movement, and whether to repeat. 

<rect x="25" y="25" width="50" height="50" style="fill:green;fill-opacity:0.4;">
	<animate attributeName="x" from="25" to="150" dur="3s" repeatCount="indefinite" /> 
</rect>

There are other animation tags, such as Animate Transform, that allow us to change other properties. This time we also used an id to apply the animation, rather than having the animation wrapped in the ellipse tag. 

<ellipse id="ellipse" cx="50" cy="70" rx="50" ry="30" style="fill:orange">
	<animateTransform xlink:href="#ellipse" attributeName="transform" attributeType="XML" type="rotate" from="0 75 75" to="360 75 75" dur="3s" repeatCount="indefinite"/> 
</ellipse>

We can start animations on events like a click. Click the rectangle to change the opacity.

<rect x="25" y="25" width="50" height="50" style="fill:green;fill-opacity:0.4;">
	<animate attributeName="opacity" from="1" to="0" dur="3s" begin="click" /> 
</rect>

Here we've added a bounce animation that happens when you hover over the shape. Note that the Animate tags can't be shared between different shapes. This is a definite downside to using SMIL tags as it leads to long and messy code. It would probably be better to use CSS for this kind of animation.

<rect x="50" y="50" width="25" height="50" style="fill:red;">
	<animate id="up1" attributeName="y" from="50" to="40" dur=".1s" begin="mouseover" /> 
	<animate attributeName="y" from="40" to="50" dur=".1s" begin="up1.end" /> 
</rect>
<rect x="80" y="50" width="25" height="50" style="fill:orange;">
	<animate id="up1" attributeName="y" from="50" to="40" dur=".1s" begin="mouseover" /> 
	<animate attributeName="y" from="40" to="50" dur=".1s" begin="up1.end" /> 
</rect>
<rect x="110" y="50" width="25" height="50" style="fill:yellow;">
	<animate id="up1" attributeName="y" from="50" to="40" dur=".1s" begin="mouseover" /> 
	<animate attributeName="y" from="40" to="50" dur=".1s" begin="up1.end" /> 
</rect>

Basic Animation with CSS


We can achieve the same basic movement with CSS. We can even separate the movement out into separate pieces.

<rect class="css_move" width="50" height="50" style="fill:rgb(100, 15, 20);">

........

.css_move {
	animation: slide 4s infinite;
}

@keyframes slide {
	0% {x: 50; y: 50 }
	50% {x: 75; y: 50 }
	100% {x: 150; y: 150}
}

We can also use CSS to change properties like color.

<svg class="css_color" width="200" height="200" >
	<circle cx="50" cy="50" r="40" />
</svg >

........

.css_color {
  fill: red;
  animation: pulse 10s infinite;
}

@keyframes pulse {
  0%   { fill: #ff0000 }
  17%  { fill: #ffff00 }
  33%  { fill: #00ff00 }
  50%  { fill: #00ffff }
  67%  { fill: #0000ff }
  83%  { fill: #ff00ff }
  100% { fill: #ff0000 }
}

Normal CSS events like hover can still be used to create animations.

<rect class="css_hover" x="25" y="25" width="50" height="50" />
		
........

.css_hover {
  fill: red;
}

.css_hover:hover {
  fill: blue;
}

A path can be animated using CSS too. As you can see, paths can become very complicated and difficult to create manaully. There are many SVG editing tools online to help create shapes and paths easily.

........

.css_path {
  stroke-dasharray: 1000;
  stroke-dashoffset: 1000;
  animation: dash 6s linear alternate infinite;
}

@keyframes dash {
  from {
    stroke-dashoffset: 1000;
  }
  to {
    stroke-dashoffset: 0;
  }
}

More Animation with SMIL


Multiple animations can be applied to the same shape within an SVG. They can animate different properties in different ways with different durations. 

<circle cx="80" cy="100" r="40" style="fill:grey;">
	<animate attributeName="r" from="30" to="70" dur="5s" repeatCount="indefinite"/> 
	<animate attributeName="cx" from="80" to="120" dur="1s" repeatCount="indefinite"/> 
</circle>

We can time when we want animations to happen based on events like clicks, literal seconds, and even other animations. Here, click the orange rectangle, and after two seconds, one animation will move it up. Then, when that is complete, the other animation will move it back down. 

<rect x="25" y="125" width="125" height="50" style="fill:orange;">
	<animate id="move_up" attributeName="y" from="125" to="25" dur="3s" begin="click + 2s" /> 
	<animate attributeName="y" from="25" to="125" dur="3s" begin="move_up.end" /> 
</rect>

With Animate Motion, we can move a shape along a specific path. This path can be simple or extremely complex.

<rect x="25" y="25" width="50" height="50" style="fill:purple;" >
	<animateMotion dur="3s" repeatCount="indefinite" path="M10,110 A120,120 -45 0,1 110 10 A120,120 -45 0,1 10,110" /> 
</rect>

You can morph between two shapes using a path or series of points. Click the shape to see the morph. For the morph animation to work, the two paths need to have the same number of points.

<polygon fill="rgb(200, 90, 40)" points="0,40, 30,40 30,25 60,25 60,40 90,40 90,190 0,190">
	<animate attributeName="points" to="30,110 0,110 40,20 80,20 50,90 90,90 0,190 0,189" begin="click" fill="freeze" dur="1s" /> 
</polygon>

SVGs can have text elements and those can also be animated.

<path id="path">
	<animate attributeName="d" from="m0,110 h0" to="m0,110 h400" dur="3s" repeatCount="indefinite" /> 
</path>
<text font-size="26" font-family="Montserrat">
	<textPath xlink:href="#path"> SVG Text. </textPath> 
</text>

Similar to moving an object along a path, we can animate text moving along a path. This path is made up of two different curves, and you can see a small inconsistency where they meet. Notice the text twitch as it moves over that spot. 

<path id="path_move" fill="none" stroke="red" stroke-miterlimit="10" d="m3.5,93.5c61.5,-51 64,67 169.5,39c85.166667,-52 140.333333,80 167.5,39" />
<text font-size="20" font-family="Montserrat">
	<textPath xlink:href="#path_move"> Text that moves along a path. 
		<animate attributeName="startOffset" from="0%" to ="100%" begin="0s" dur="8s" repeatCount="indefinite" keyTimes="0;1" calcMode="spline" keySplines="0.1 0.2 .22 1"/>
	</textPath> 
</text>

We can use layers to our advantage in order to give depth to animations. Here, we have a blue rectangle with a green square rotating on top of it. We then put another half a rectangle on top of that, in black. This gives the impression that the green square is going around the rectangle. 

<rect x="50" y="10" height="150" width="50" fill="blue" />
<g transform=" matrix(0.866, -0.5, 0.25, 0.433, 80, 80)">
	<rect x="40" y="40" height="20" width="20" fill="lime">
		<animateTransform attributeName="transform" type="rotate" from="360 0 0" to="0 0 0" dur="1s" repeatCount="indefinite" />
	</rect>
</g>
